Overview

The goptions framework provides command line and/or JSON file parsing of user defined options.
The JSON files (steering cards) have the extension ".jcard" and are referred to as jcards.

Switch

A switch is a boolean variable, initialized to "false", that is switched true by command line.
For example, the switch "gui" is true if the command lne contains:

  -gui

Simple Option

A simple option has a int, float, double or string value associated with it, and looks like this in the jcard:

"runno": 12

and the corresponding command line option is:

  -runno=11

Structured Option

A structured option has multiple objects associated with a tag. In the jcard:

"beam": {
"vertex": "0, 0, -3, cm",
"value": "electron, 4*GeV, 20*deg, 180*deg"
}

The corresponding command line option is:

  -beam={ vertex: 0, 0, -3, cm; mom: electron, 4*GeV, 20*deg, 180*deg; }

Notice the absence of quotes in the command line: the colons delimit the values, the semi-colons delimit the fields.

Goptions Main Features

  • Goptions are dynamically added to an executable by a framework or plugin.
  • Superposition of command line / jcards option values.
  • Mechanism to project options onto user defined structure/class.
  • Mechanism to import child jcards.
  • JSON output of user selections.
  • HTML output of options help.

C++ User Interface

The users build his/her own GOptions class by calling the constructor: 

 GOptions(argc, argv, defineOptions())
Parameters
argc, argvpassed from "main"
defineOptions()is the function building the options map.
